Case Study 2

Project Horizon: Introducing "Pay Now, Start Later"

This project involved identifying a key user need for flexibility in membership start dates. I'll walk you through how we used data to build a business case for a new feature that captured previously lost revenue.

The User Need

Through analysis of user feedback and sign-up data, I identified a clear pattern: many potential members wanted to take advantage of promotional offers but weren't ready to start their membership immediately. This created a significant friction point, leading to lost conversions.

Key Findings:

  • High drop-off rates during promotional periods from users who hesitated at payment.
  • Qualitative feedback from customer support confirmed demand for flexible start dates.
  • Cohort analysis suggested a large, untapped segment of "plan-ahead" users.

The Solution: Pay Now, Start Later

We introduced a new option in the sign-up flow allowing users to select a membership start date up to 30 days in the future. This simple change had a powerful impact.

Billing Process:

1

User pays for the first month and any joining fees immediately.

2

The chosen future start date is recorded.

3

Membership is activated on the start date, granting gym access.

4

Subsequent billing cycles align with the chosen start date.
